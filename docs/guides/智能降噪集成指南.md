# 智能降噪集成指南

## 功能概述

智能降噪系统通过 **消息去重** 和 **消息合并** 两步，有效减少 DevOps 通知的消息数量。

---

## 第一步：消息去重（已完成）

### 功能说明
- 基于消息内容哈希，在短时间内屏蔽重复消息
- 去重窗口：默认 5 分钟（可配置）
- 适用场景：Jenkins 构建失败、监控告警、JIRA 状态变更等

### 配置项
```yaml
# 消息去重配置
notification.dedup.window-seconds: 300  # 5 分钟
```

### 使用方式
`NotificationService.sendNotification()` 会自动调用 `MessageDedupService.isDuplicate()` 进行去重检查。

---

## 第二步：消息合并（已完成）

### 功能说明
- 将短时间内的同类消息合并为一条摘要推送
- 合并窗口：默认 5 分钟（可配置）
- 合并阈值：默认 5 条（达到立即推送）
- 适用场景：CI/CD 构建、监控告警、JIRA 更新等

### 配置项
```yaml
# 消息合并配置
notification.batch.enabled: true
notification.batch.window-seconds: 300    # 5 分钟
notification.batch.threshold: 5            # 达到 5 条立即推送
```

### 合并摘要格式
```
📊 **构建事件汇总**（过去 5 分钟）

1. Jenkins 构建失败（重复 3 次）
2. 单元测试失败

💡 共 5 条 BUILD 事件
```

## 集成方式

### 1. 在 GitHubWebhookService 中集成（已完成）
`GitHubWebhookService` 已注入 `NotificationService`，所有 Webhook 通知都会经过智能降噪。

### 2. 在其他服务中集成（待完成）
如果有其他 DevOps 客户端（如 Jenkins、JIRA、监控），可按以下方式集成：

```java
@Autowired(required = false)
private NotificationService notificationService;

// 发送通知（带智能降噪）
notificationService.sendNotification(chatId, "BUILD", "Jenkins 构建失败");
```

## 配置示例

```yaml
# 通知配置（智能降噪）
notification:
  noise-reduction-enabled: true  # 启用智能降噪

# 消息去重配置
notification.dedup.window-seconds: 300

# 消息合并配置
notification.batch.enabled: true
notification.batch.window-seconds: 300
notification.batch.threshold: 5
```

---

## 测试建议

1. **去重测试**：短时间内发送相同内容的消息，应只收到一条
2. **合并测试**：发送 5 条同类消息，应收到合并摘要
3. **紧急通知测试**：使用 `sendUrgentNotification()` 跳过合并，但仍去重

---

## 监控指标

- 去重拦截率：`Redis Key notify:dedup:*` 的数量
- 合并队列长度：`Redis Key notify:batch:queue:*` 的长度
- 推送延迟：合并窗口时间（默认 5 分钟）

---

## 故障排查

### 1. 消息未推送
- 检查 Redis 是否可用
- 检查 `notification.noise-reduction-enabled` 是否为 true
- 查看日志中的 "消息已被去重拦截" 或 "消息已加入合并队列"

### 2. 消息延迟过长
- 减小 `notification.batch.window-seconds`
- 减小 `notification.batch.threshold`

### 3. 合并摘要未生成
- 检查 `MessageBatchScheduler` 是否正常运行（每 60 秒执行一次）
- 查看日志中的 "合并消息已推送"
