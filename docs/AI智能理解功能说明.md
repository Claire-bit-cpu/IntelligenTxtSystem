# AI 智能理解上下文功能说明

## 功能概述

AI智能理解功能允许飞书机器人在群聊中理解用户的自然语言消息，并根据上下文自动执行相应操作或回复对话。

## 工作原理

### 四级响应策略

1. **Level 1 (精确匹配)**：`/`开头 + 指令存在 → 正常执行
   - 例如：`/weather 北京` → 执行天气查询

2. **Level 2 (模糊匹配)**：`/`开头 + 指令不存在但相似度达标 → 提示"您是不是想说 /xxx？"
   - 例如：`/weathr` → 提示"您是不是想说 /weather？"

3. **Level 3 (静默丢弃)**：非`/`开头且未@机器人 → 返回 null（不回复）
   - 普通群聊消息，机器人不响应

4. **Level 4 (AI 理解)**：@机器人 + 非`/`开头 → 调用 AI 理解意图后处理
   - 例如：`@机器人 今天天气怎么样` → AI识别为天气查询指令并执行

### AI理解的三类意图

- **command**：执行命令
  - AI识别用户想执行某个指令，自动转换为`/指令`执行
  - 例如："帮我查一下北京天气" → 执行`/weather 北京`

- **chat**：直接回复对话
  - AI直接生成回复，不执行指令
  - 例如："你好" → "你好！我是智能助手，有什么可以帮您？"

- **clarify**：追问（信息不足）
  - AI认为信息不足，向用户追问
  - 例如："帮我查天气" → "请问您想查哪个城市的天气？"

## 使用方法

### 1. 在群聊中使用

在群聊中@机器人，然后输入自然语言消息：

```
@智能助手 今天天气怎么样
@智能助手 帮我看看最新的代码提交
@智能助手 我想创建一个JIRA任务
```

### 2. 查看对话历史

机器人会记住最近的对话历史（最多50条），可以用于多轮对话：

```
用户：@智能助手 北京天气怎么样
机器人：北京今天晴天，温度25-30℃
用户：@智能助手 那上海呢？
机器人：上海今天多云，温度23-28℃
```

### 3. 清空对话历史

如果需要重新开始对话，可以使用`/clear_history`指令：

```
/clear_history
```

### 4. 测试AI理解功能

可以使用`/test_ai`指令测试AI是否能正确理解消息：

```
/test_ai 我想查天气
```

## 配置说明

在`application.yaml`中可以配置AI智能理解功能：

```yaml
# AI 智能理解配置
ai:
  enabled: true  # 是否启用AI理解功能
  system-prompt: ""  # 系统提示词（可选，用于定制AI理解行为）
```

也可以通过环境变量配置：

```bash
AI_ENABLED=true
AI_SYSTEM_PROMPT="你是一个专业的智能助手"
```

## 技术实现

### 核心组件

1. **AiUnderstandingService**
   - AI理解核心服务
   - 负责调用通义千问API进行意图识别
   - 管理对话历史存储（Redis）

2. **MessageDispatcher（修改）**
   - 新增Level 4处理逻辑
   - 判断消息@机器人且非`/`开头时，转发给AiUnderstandingService处理

3. **MessageProcessor（修改）**
   - 新增对话历史落库逻辑
   - 每次处理消息后将用户消息、机器人回复存入对话历史

4. **ContextManager（修改）**
   - 新增对话历史存储能力（Redis List存储）
   - Key为`ctx:history:{chatId}:{userId}`，最多50条，7天过期

### 数据存储

- **对话历史**：存储在Redis中，Key格式为`ctx:history:{chatId}:{userId}`
- **用户上下文**：存储在Redis中，Key格式为`ctx:global:{userId}`和`ctx:local:{userId}:{contextType}`

## 注意事项

1. **API Key配置**：需要确保通义千问API Key已正确配置（`qianyu.api-key`）
2. **Redis配置**：需要确保Redis已正确配置，用于保存对话历史
3. **网络环境**：需要确保服务器能够访问通义千问API
4. **消息长度限制**：建议用户消息不超过1000个字符，避免API调用失败

## 故障排查

### AI理解功能不工作

1. 检查`ai.enabled`配置是否为`true`
2. 检查通义千问API Key是否正确配置
3. 检查Redis是否正常运行
4. 查看日志文件，确认是否有错误信息

### 对话历史不保存

1. 检查Redis是否正常运行
2. 检查Redis Key是否正确（`ctx:history:{chatId}:{userId}`）
3. 查看日志文件，确认是否有保存失败的错误信息
